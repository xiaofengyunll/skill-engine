# Security Checklist

当变更涉及身份、权限、用户输入、持久化、文件、页面渲染、回调、依赖、密钥、日志、多租户数据或公开接口时使用本清单。

合并/发布结论统一按 `risk-assessment-template.md` 的 Merge Decision Rules 裁决。本文件只提供安全专项审查项和高危样例。

## 认证与授权

- 接口、页面、任务、管理工具、回调和内部 API 具备正确认证。
- 角色校验、对象级权限、数据范围和租户隔离执行一致。
- 授权不只依赖前端、网关或调用方。
- 权限提升、水平越权和跨租户访问已有测试或人工验证。

## 输入、注入与 Web 安全

- 请求体、查询参数、请求头、文件、事件、回调和导入数据已校验。
- SQL/NoSQL、命令、路径穿越、模板、LDAP、表达式语言和 SSRF 风险已控制。
- XSS、CSRF、CORS、Cookie 安全、开放重定向和不安全 HTML 渲染已处理。
- 反序列化、动态类加载、表达式求值和插件/脚本执行被避免或受控。

## 密钥与敏感数据

- 密钥不会被提交、记录日志、返回给客户端、暴露在错误信息中或打入前端包。
- 日志、指标、审计、导出、链路追踪和异常中的敏感字段已脱敏。
- Token 的过期、轮换、作用域、存储和撤销策略合理。
- 生产路径关闭调试接口和过度详细的错误响应。

## 依赖与供应链

- 新增或升级依赖的漏洞和许可证风险可接受。
- 构建脚本、生成代码、postinstall hook 和外部下载来源可信，必要时锁定版本。
- 容器镜像、基础镜像和运行时包没有引入已知关键漏洞。

## 文件、回调与多租户数据

- 上传类型、大小、存储路径、预览/渲染、扫描和下载授权已控制。
- Webhook/callback 的签名、时间戳、防重放、白名单和失败行为已验证。
- 数据库查询、缓存、索引、队列、异步任务和导出均应用租户 ID 和数据范围。

## 安全专项高危样例

以下问题通常应判为致命或数据/安全类高风险，并按 `risk-assessment-template.md` 的合并规则处理：
- 可访问受保护数据或执行受保护操作的越权路径。
- 租户数据泄露或缺少对象级授权。
- 源码、日志、响应或前端资源中暴露密钥。
- 可实际利用的注入、XSS、文件穿越、SSRF、反序列化或命令执行路径。
- 公开 API 或回调在无签名、无授权的情况下接受可信动作。
