## 架构演进建议

按三个时间范围给出建议：

- 立即处理：发布或近期安全性。
- 下一迭代：降低变更成本的模块/分层清理。
- 长期治理：治理规则、契约、文档和重构路线。

每条建议必须说明改善点和引入的权衡。
