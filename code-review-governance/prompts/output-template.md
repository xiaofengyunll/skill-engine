# Code Review Governance Output Template

## 使用模式

- 快速 PR Review：用于普通 PR、局部变更或用户只需要合并建议时。直接使用 `quick-review-template.md`，不要在本文件维护第二份快速模板。
- 深度架构 Review：用于仓库级、跨模块、架构治理、重构评估或高风险变更。使用下面的完整模板。

## 审查上下文

- 审查模式：`团队 PR / 个人自审`
- 审查入口：`本次 diff / 某个模块 / 整个仓库 / bugfix 回归`
- 结论级别：`完整审查 / 条件审查 / 静态预审`

- 明确排除或降权范围：

## 总体工程质量评估

- 架构质量：`优秀 / 良好 / 可接受 / 存在明显风险 / 不建议继续扩展`
- 可维护性：`优秀 / 良好 / 可接受 / 偏弱 / 高风险`
- 可扩展性：`优秀 / 良好 / 可接受 / 偏弱 / 高风险`
- 技术债务风险：`低 / 中 / 高 / 极高`
- 模块边界健康度：`清晰 / 基本清晰 / 局部模糊 / 严重混乱`
- 协作/自维护友好度：`高 / 中 / 低`

简要结论：

> 用 3-6 句话说明当前实现的工程状态、主要风险、是否适合继续扩展、是否建议立即治理。

## 核心发现

### 1. `[风险等级]` 问题标题

- 问题位置：`path/to/file:line` 或模块/服务/接口名称
- 问题原因：说明直接原因，避免只写现象
- 为什么这是问题：说明它违反了需求、设计、架构边界、职责划分、可维护性或协作规则中的哪一类
