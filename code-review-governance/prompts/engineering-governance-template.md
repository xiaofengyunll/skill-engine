## Review 门禁规则

### 准入条件

- 需求/任务/设计证据：
- CI/build/lint/test 状态：
- API/schema/message 兼容性说明：
- 迁移和回滚说明：
- 安全影响说明：
- 测试计划和已执行命令：

### 准出条件

- 无致命问题：
- 普通高风险已修复，或已按 `risk-assessment-template.md` 的 Approval Matrix 完成豁免：
- 数据/安全/兼容/核心链路高风险已修复，或具备发布级批准和完整豁免记录：
- 致命问题原则上不接受豁免；例外必须按 Approval Matrix 记录发布级风险：
- 必要测试/验证已完成：
- 豁免均包含 owner、批准人、截止时间、backlog 和关闭条件：
- 发布/回滚风险可接受：

## 团队协作审查

评估：
- 新成员能否在一小时内理解模块归属。
- 多名工程师能否并行修改相邻功能，而不频繁产生冲突或归属争议。
- 约定能否从文档和代码中被发现。
- 日志和异常是否有助于事故响应。
- Code Review 标准是否可执行，而不是完全依赖主观判断。
